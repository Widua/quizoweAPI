Pomysł na projekt

Na pomysł projektu wpadłem gdy usłyszałem o międzyszkolnym festiwalu informatycznym "projekt na szóstkę", postanowiłem że zrobię coś co będzie łączyło prawie każdy typ projektu, czyli API bez którego zarówno strona internetowa, aplikacja mobilna jak i część aplikacji desktopowych nie może w pełni funkcjonować. A więc swego rodzaju łącznik dla każdego typu aplikacji z serwerem i bazą danych, z pomocą tego każdy użytkownik internetu który pobierze ten program, może uruchomić ten serwer, a następnie podpiąć go do własnego projektu np. Strony internetowej czy Aplikacji mobilnej.
W przyszłości chciałbym też uczynić ten program projektem opensource, ponieważ sam korzystam z wielu takich na codzień, i uważam że opensource to przyszłość!

Wybór technologii

W wyborze technologii przede wszystkim myślałem o tym, aby projekt działał na każdym urządzeniu, dlatego wybrałem Javę która jest do uruchomienia tak samo na Windowsie jak i na Linuxie oraz MacOS. Do zarządzania API wybrałem technologię Java Spring, która jest javovym frameworkiem do backendu webowego, wykorzystałem takie komponenty jak: Spring Data, Spring Security, Spring Web, Spring-MongoDB, Spring Validation
Jako bazę danych wybrałem MongoDB, która jest nierelacyjną bazą danych opartą na JSONie, ponieważ jest prostsza w obsłudze w porównaniu do np. MySQL czy PostgreSQL co spełnia ideę projektu, czyli to aby każdy użytkownik internetu mógł bez problemu API obsłużyć i zaimplementować w swoim projekcie.
Do stworzenia tej małej dokumentacyjno-poradnikowej strony użyłem technologii Thymeleaf, która zapewnia proste w obsłudze obsługujące HTML5 funkcje. Wizualna część strony została wykonana w HTML5 oraz CSS, do podstrony umozliwiającej testowanie API wykorzystałem język JavaScript.
Hasła użytkownikow są hashowane za pomocą algorytmu BCRYPT, przez co hasło jest ciężej złamać.
